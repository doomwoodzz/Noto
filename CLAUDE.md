## Project Overview

Noto is a native macOS SwiftUI prototype for a local-first Markdown notes workspace with an AI lecture-listening assistant. It features a three-pane desktop window (vault sidebar, editor, context panel), wiki-link support with generated backlinks, a "Knowledge Web" graph view, a simulated AI recorder, and a command palette.

Slogan: "When you listen, Noto remembers."

## Tech Stack

- SwiftPM package with Swift 6 language mode
- SwiftUI for UI, AppKit for the custom `NSTextView`-based Markdown editor
- `@Observable` (Observation framework) for reactive state
- Sparkle framework for auto-updates
- No backend; in-memory mock vault data

## Branch Structure

- **`main`**: Contains design specs, implementation plans, and docs in `docs/superpowers/`. Minimal source files.
- **`noto-implementation`**: Contains the full SwiftPM package with all source code. Active development happens here.
- Worktrees are used for isolated development (`.worktrees/noto-implementation/`).

## Build, Test, and Run

```bash
# Build debug
swift build

# Build release executable
swift build -c release --product Noto

# Run tests (uses Swift Testing, not XCTest)
swift test

# Run a single test filter
swift test --filter MarkdownParserTests

# Run the app
swift run Noto

# Run the check harness (executable alternative to tests)
swift run NotoCoreChecks

# Package release DMG (requires release build first)
zsh scripts/package-noto-dmg.sh
```

## Architecture

### Targets

| Target | Type | Purpose |
|--------|------|---------|
| `NotoCore` | Library | Pure Swift models, parsers, metadata/graph builders, note actions. No UI. |
| `Noto` | Executable | SwiftUI app entry, views, `@Observable` app state, keyboard shortcuts. |
| `NotoCoreChecks` | Executable | Runtime check harness that exercises core logic without a test runner. |
| `NotoCoreTests` | Test | Swift Testing tests with unsafe linker flags for CommandLineTools frameworks. |

### Key Source Directories

- `Sources/NotoCore/Models/` — `Vault`, `VaultFile`, `MetadataCache`, `FileMetadata`, `GraphNode`, `GraphEdge`, `LectureMemory`, `RecorderPhase`
- `Sources/NotoCore/Lib/` — `MarkdownParser`, `MarkdownEditor`, `MetadataCacheBuilder`, `GraphBuilder`, `NoteActions`, `AIRecorderModel`, `WorkspaceStore`
- `Sources/NotoCore/Data/` — `MockVault.school` with seeded School Vault notes
- `Sources/Noto/Views/` — SwiftUI views: `MacWindowView`, `VaultSidebarView`, `MarkdownWorkspaceView`, `MarkdownPreviewView`, `MarkdownEditorView`, `KnowledgeGraphView`, `CommandPaletteView`, `AIRecorderPanelView`, `AudioWaveformView`
- `Sources/Noto/` — `NotoApp.swift`, `AppState.swift`, `UpdaterController.swift`
- `Checks/NotoCoreChecks/` — Executable check harness entry point

### State Flow

1. `AppState` (SwiftUI `@Observable`) wraps `WorkspaceStore` and holds presentation state (command palette, recorder).
2. `WorkspaceStore` (value type, testable) holds `Vault`, active file/tab, search query, and recorder model.
3. `WorkspaceStore.metadata` is computed via `MetadataCacheBuilder.build(files:)` — backlinks are generated by resolving outgoing wiki links across all files.
4. `WorkspaceStore.graph` is computed via `GraphBuilder.build(files:cache:)`.
5. `WorkspaceStore.visibleGraph` applies the active `GraphFilter` (all, local, lectureOnly, orphans).
6. The Markdown editor uses a custom `NSViewRepresentable` wrapping `NSTextView` with `MarkdownEditor` utilities for inline formatting, list continuation, and tab indentation.

### Keyboard Shortcuts

Bound in `NotoApp.swift` via `.commands`:
- `Cmd+K` — Toggle Command Palette
- `Cmd+Ctrl+M` — Toggle AI Recorder
- `Cmd+B/I/U` — Bold/italic/underline in editor (handled by `KeyHandlingTextView`)

### Markdown Parsing

`MarkdownParser` extracts from raw Markdown:
- Wiki links: `[[Title]]`
- Headings: `# Heading`
- Tags: `#tag` (excludes heading lines)
- Word count (excluding markup)
- Checklist items: `- [ ]` / `- [x]`

Backlinks are always derived, never authored directly.

## Release and Packaging

- **GitHub Actions** (`.github/workflows/release.yml`): Triggered on `v*.*.*` tags. Builds release executable, packages DMG, generates Sparkle appcast, uploads release assets, and commits `appcast.xml` back to `main`.
- **DMG Script** (`scripts/package-noto-dmg.sh`): Creates a signed `.app` bundle with `Info.plist`, embeds Sparkle framework, produces a compressed DMG in `dist/`.
- **Sparkle**: `UpdaterController` checks for `SUFeedURL` and `SUPublicEDKey` in the bundle at runtime. Unconfigured builds show "Updates Unconfigured" in the UI.

## Testing Notes

- The test target uses `unsafeFlags` for CommandLineTools framework/library paths because the local environment provides Swift Testing but not XCTest.
- Tests cover: parser, metadata builder, graph builder, note actions, recorder model, app state behavior, and mock vault invariants.
- The `NotoCoreChecks` executable provides a runtime check harness that runs the same assertions without a test runner.
